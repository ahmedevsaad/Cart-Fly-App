# CartFly Bug Audit — 2026-06-13 (75+ findings, 7 parallel audits)

## Batch A — async / Firestore resilience & crashes
1. plan_detail_screen: `await subscribe('basic')` before nav can hang → fire-and-forget.
2. PlanProvider.subscribe: no timeout/catch; writes REAL Firestore even in demo (uid demo-user) → add `demo` flag + timeout + catch.
3. OrderRepository.advance/getOnce: `doc.data()!` null-deref; no timeouts → null-guard + `.timeout`.
4. OrderRepository.create: no timeout → `.timeout`.
5. track_order_screen: FutureBuilder `getOnce` fires new future every build → StatefulWidget + cached future.
6. lockersByCode (data/lockers.dart): firstWhere no orElse → crash on bad code.
7. AuthProvider/OrdersProvider(demo microtask)/SettingsProvider: notifyListeners after dispose → `_disposed` guards.
8. seedSampleIfEmpty: not idempotent/atomic, no timeout → `seeded` flag guard.

## Batch B — navigation
9. login/register back `context.pop()` on empty stack (reached via go) → canPop fallback / go(welcome).
10. payment_error `context.pop()` dead-end if deep-linked → canPop fallback to /home.
11. how_it_works "Next" `context.go(home)` destroys Settings stack → `context.pop()` when poppable.
12. create_shipment success `context.go(track)` drops back stack/nav → `context.push`.
13. settings "Saved Addresses" row has no onTap (dead) → wire to myAddress.
14. confirm_order null orderId → empty `for=` → payment completes with nothing → guard.
15. order_detail "Proceed to checkout" shown even when delivered → hide for terminal status.

## Batch C — forms / validation / logic
16. register: password-mismatch check before empty check (empty==empty passes) → reorder.
17. register: no email format validation → add basic check.
18. register/change_password: passwords trimmed (alters actual password) → don't trim password values.
19. create_shipment: submits all-empty → require merchant or tracking #.
20. create_shipment: category stored as `.label` not `.name` (breaks pricing match) → use enum name.
21. create_shipment: Est value keyboard `number` → `numberWithOptions(decimal:true)`.
22. calculator: weight dialog zero/negative no feedback; from==to silent wrong rate → guard + message.
23. pricing: missing country pairs fall back to 4.8 silently → extend table for all UI pairs.
24. verify: wrong-length code no feedback → inline error.
25. track_order: `delivered` maps to step 2 not 3.
26. orders demo create: colliding ids (`demo${len+1}`) → monotonic counter.
