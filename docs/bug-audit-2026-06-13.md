# CartFly Bug Audit — 2026-06-13 (75+ findings, 7 parallel audits)

## Batch A — async / Firestore resilience & crashes
1. plan_detail_screen: `await subscribe('basic')` before nav can hang → fire-and-forget.
2. PlanProvider.subscribe: no timeout/catch; writes REAL Firestore even in demo (uid demo-user) → add `demo` flag + timeout + catch.
3. OrderRepository.advance/getOnce: `doc.data()!` null-deref; no timeouts → null-guard + `.timeout`.
4. OrderRepository.create: no timeout → `.timeout`.
5. track_order_screen: FutureBuilder `getOnce` fires new future every build → StatefulWidget + cached future.
6. lockersByCode (data/lockers.dart): firstWhere no orElse → crash on bad code.
7. AuthProvider/OrdersProvider(demo microtask)/SettingsProvider: notifyListeners after dispose → `_disposed` guards.
8. seedSampleIfEmpty: not idempotent/atomic, no timeout → `seeded` flag guard.

## Batch B — navigation
9. login/register back `context.pop()` on empty stack (reached via go) → canPop fallback / go(welcome).
10. payment_error `context.pop()` dead-end if deep-linked → canPop fallback to /home.
11. how_it_works "Next" `context.go(home)` destroys Settings stack → `context.pop()` when poppable.
12. create_shipment success `context.go(track)` drops back stack/nav → `context.push`.
13. settings "Saved Addresses" row has no onTap (dead) → wire to myAddress.
14. confirm_order null orderId → empty `for=` → payment completes with nothing → guard.
15. order_detail "Proceed to checkout" shown even when delivered → hide for terminal status.

## Batch C — forms / validation / logic
16. register: password-mismatch check before empty check (empty==empty passes) → reorder.
17. register: no email format validation → add basic check.
18. register/change_password: passwords trimmed (alters actual password) → don't trim password values.
19. create_shipment: submits all-empty → require merchant or tracking #.
20. create_shipment: category stored as `.label` not `.name` (breaks pricing match) → use enum name.
21. create_shipment: Est value keyboard `number` → `numberWithOptions(decimal:true)`.
22. calculator: weight dialog zero/negative no feedback; from==to silent wrong rate → guard + message.
23. pricing: missing country pairs fall back to 4.8 silently → extend table for all UI pairs.
24. verify: wrong-length code no feedback → inline error.
25. track_order: `delivered` maps to step 2 not 3.
26. orders demo create: colliding ids (`demo${len+1}`) → monotonic counter.

## Round 2 audits (5 more agents) — ~85 additional findings → 150+ total

### States / empty / loading / edge
- OrdersProvider stream has no onError → dead stream looks like empty account.
- orders_screen: no loading state (shows empty for 1-4s before stream fires).
- home avatar: use initials fallback not generic icon.
- order_detail "Order not found": hardcoded EN, no back/retry.
- create_shipment: locker method with empty lockers → order created with null lockerId, no validation.
- track_order trackExpectedDate static (not from createdAt).
- profile/edit_profile: country stored as ISO 'eg' but compared to display 'Egypt' → flag null / silent reset on save.
- confirm_order: empty phone/name '—' read-only, no edit link.
- plan_detail: unknown plan code renders raw code, empty desc/price.
- home tracker activeIndex hardcoded 1 (not from real latest order).
- payment form pre-filled demo card for ALL users (gate behind kDebugMode/bypass).
- create_shipment: no qty/value range validation.

### RTL / Arabic
- cf_status_timeline connectors use Positioned left/right (not directional) → wrong side in RTL.
- order_detail _HubRow Icons.chevron_right_rounded (no RTL mirror).
- how_it_works title/Next hardcoded EN; _StepRow ordering.
- calculator currency '$' prefix + 'Days' hardcoded, not intl/localized.
- profile/edit/home/orders/payment crossAxisAlignment + price columns not direction-aware.
- CfIcons.arrowRight / signOut don't flip in RTL.
- create_shipment hintText hardcoded EN; login_screen 'Sign in to continue'/'Login'/'Register' hardcoded.

### Boundaries / perf
- pricing: NaN/Infinity weight → FormatException; no upper bound (1e9 → billions).
- calculator weight dialog: only v>0, no upper bound.
- orders_screen id substring guard `>4` off-by-one.
- register currency split(' — ').first fragile.
- warehouse/locker firstWhere orElse hides missing data (assert in debug).
- track_order getOnce extra fetch even when in stream.
- home GridView childAspectRatio fixed → overflow at large text scale.
- GoogleFonts.inter called per-build in package cards (cache).
- floating-point equality in weight decimal formatting.

### Security / data-integrity (CRITICAL)
- firestore.rules: `verified`, `plan`, `status` (orders) are CLIENT-WRITABLE → bypass OTP, free Prime, self-mark delivered. Restrict writable fields.
- OTP_BYPASS defaults '000000' in all builds (demo feature — keep but document/guard for release).
- payment advance() on arbitrary order URL param (Firestore-scoped so safe, but swallowed StateError).
- seedSampleIfEmpty TOCTOU race → duplicate seeds (use transaction).
- resendCode no rate limit; OTP no brute-force lockout.
- plan_provider.subscribe accepts arbitrary code (no allowlist).
- DEBUG_BYPASS_AUTH could ship if CI sets it (assert kReleaseMode).

### Consistency / dead code / coverage
- order_detail _HubRow uses Material Icons instead of CfIcons (homeBuilding/lockers/plans/chevronRight).
- calculator country-picker check Icons.check vs CfIcons.stepCheck.
- about/support/profile/settings use raw GoogleFonts.inter instead of AppText.* (worst offenders).
- payment_success uses planFree (green) on a Prime (purple) receipt.
- plan_detail/create_shipment use Colors.transparent / theme.colorScheme instead of tokens.
- flutter_map dep + LatLng data unused (dead weight) — remove or use.
- ~30 unused assets in assets/images/ (old html-clone bg pngs) inflate bundle.
- CfIcons.star unused.
- 13 screens have no widget tests.
- payment checkout hardcodes 'Prime cart'/'$19.99' regardless of selected plan.
